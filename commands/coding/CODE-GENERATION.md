# Code Generation

## Overview
Generate high-quality, production-ready code based on specifications with best practices and proper error handling.

## Instructions
Generate code that follows these principles:
1. Write clean, readable, and maintainable code
2. Follow language-specific best practices and conventions
3. Include proper error handling and edge cases
4. Add type hints/annotations where applicable
5. Use meaningful variable and function names
6. Keep functions focused and single-purpose

## Checklist
- [ ] Code follows project coding standards
- [ ] Proper error handling implemented
- [ ] Edge cases considered
- [ ] Variables and functions have descriptive names
- [ ] Code is modular and reusable
- [ ] Performance considerations addressed
- [ ] Security best practices followed
- [ ] Comments added for complex logic

## Output Format
- Include file path and structure
- Add inline comments for complex sections
- Provide usage examples if applicable
