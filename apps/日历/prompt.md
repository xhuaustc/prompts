**创建一个具有令人惊艳自然背景的高端日历应用（可直接落地实现），满足以下规格：**

### 目标体验
- 氛围：宁静、高效、高端、现代
- 可读性：在全屏自然背景上依然清晰易读（文字/网格/事件块层级明确）
- 交互：顺滑、有轻微动效反馈，但不花哨

### 技术与约束（硬性）
- 使用 Next.js + TypeScript + Tailwind CSS
- 全程使用 Lucide React 图标（不使用其他图标库）
- 使用 `useState` / `useEffect` 进行状态管理（不引入 Redux/Zustand）
- 代码与注释必须为英文（UI 文案可以是中文或英文，二选一保持一致）
- 使用响应式网格布局实现日历结构
- 事件块定位必须基于时间段计算，视觉位置与时间对应
- 如果用户提供了参考图：配色与布局对齐参考图；如果没有参考图：以“高端玻璃拟态 + 自然背景”自洽落地

### 视觉设计
- 全屏自然山景背景图（来源：Unsplash；建议通过 `next/image` 展示，覆盖整个 viewport）
- 玻璃拟态：大量使用 `backdrop-blur` + 半透明白色覆盖层（如 `bg-white/10`, `bg-white/20`）
- 圆角：统一现代圆角（例如 16px/20px 量级），微妙阴影与描边贯穿（`shadow`, `ring-white/10`）
- 入场动画：不同区域使用交错延迟的淡入/上移动画（轻量、统一节奏）

### 布局与结构
- 顶部 Header（简洁、单行）包含：
  - 应用标题
  - 搜索栏（可输入关键字过滤事件标题/地点/参与者）
  - 设置按钮（仅 UI）
  - 用户头像（仅 UI）
- 左侧 Sidebar（固定宽 264px，玻璃拟态）包含：
  - 蓝色“创建”按钮（带加号图标）
  - 迷你日历小部件（展示当月，支持点击某天跳转）
  - “我的日历”列表（每项带彩色指示点，可开关显示）
- 主区域：
  - 默认周视图（Week view）
  - 顶部视图切换：日 / 周 / 月
  - 导航控件：今天按钮 + 上/下一个（周或月）+ 当前日期范围显示
  - 主日历网格：清晰的时间刻度与列（周视图为周一到周日或周日到周六，任选其一但保持一致）

### 日历功能（必须实现）
- 周视图时间范围：08:00 - 16:00
- 时间刻度：至少每小时一行（可选：半小时网格线更精细）
- 事件块：
  - 颜色：多种亮色（绿/紫/黄/粉/青等），与背景形成对比
  - 位置：根据开始时间与结束时间计算 `top/height`（或 grid row span），必须准确
  - 内容：标题 + 时间（可选：地点）
- 示例事件（至少 3 个）：
  - Team Meeting
  - Client Call
  - Lunch Date
- 事件详情弹窗（Modal）：
  - 点击事件块打开
  - 展示：时间、地点、参与者、描述
  - 支持关闭（点击遮罩/关闭按钮/ESC）

### 交互元素（必须实现）
- Hover：按钮/可点击元素有顺滑悬停反馈（亮度/描边/阴影轻微变化）
- 搜索：实时过滤主视图中的事件块（无事件时给出空状态提示）
- AI 助手弹窗：
  - 页面加载后 3 秒出现
  - 有“打字动画”效果（逐字呈现一段建议）
  - 可关闭/最小化（任选其一，但要可控）
- Hans Zimmer 音乐建议模块：
  - 展示一条建议（例如：Focus playlist / Time 等）
  - 提供播放/暂停按钮（允许用模拟状态，不强制真实音频资源；如果实现真实音频，需考虑静音/自动播放限制）

### 配色方案（建议）
- 主色：`blue-500` / `blue-400`
- 文本：白色分层（`text-white`, `text-white/80`, `text-white/60`）
- 容器：半透明白（`bg-white/10`, `bg-white/20`）+ `backdrop-blur`
- 分隔线/网格：`border-white/10` 或 `ring-white/10`

### 动画与过渡（建议实现）
- 页面入场：Header / Sidebar / Main 区域 staggered fade-in
- Modal：淡入 + 轻微缩放
- AI 助手打字：每 30–60ms 增加一个字符（根据体验微调）
- 所有动画时长统一（例如 200–350ms），避免突兀

### 可访问性与细节（加分项）
- 重要按钮有可见的 focus 样式（键盘可用）
- Modal 打开时锁定背景滚动
- 颜色对比足够，事件块在浅色/亮色背景下依然清晰

### 交付物
- 可运行的页面（单页即可），视觉与交互完整
- 代码结构清晰（组件拆分合理），便于后续扩展真实数据源
